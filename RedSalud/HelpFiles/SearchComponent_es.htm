Búsquedas


Este sistema provee un componente de búsqueda en diferentes lugares. Este componente de búsqueda es un buscador muy poderoso, parecido a Google, donde el usuario debe normalmente encontrar lo que está buscando. Además le permite utilizar operadores para limitar el conjunto de resultados.

Como se puede ver en la figura de la izquierda, todo lo que tiene que hacer es escribir el criterio de búsqueda en la caja de texto del componente. El sistema intentará encontrar uno o más registros que satisfagan el criterio de búsqueda y aplicará la búsqueda en todos los campos de textos donde la misma tenga sentido. Por ello, solamente escriba una palabra o un conjunto de palabras y el sistema deberá encontrar los registros correctos.


Si está disponible, el sistema de búsqueda le permitirá especificar una búsqueda refinada en un formulario de fácil llenado. Como se muestra en la figura más abajo, haga clic en "Búsqueda Avanzada" y el sistema abrirá un formulario donde usted puede especificar los parámetros para el criterio de búsqueda.

 

Si usted desea tener mayor control sobre su búsqueda, usted puede utilizar la funcionalidad más avanzada del componente de búsqueda y escribir una expresión de búsqueda que especifica en detalle los valores que usted desea buscar para cada una de las variables disponibles en la búsqueda. Usted puede listar los campos disponibles haciendo clic en la herramienta de selección de variables:

La Herramienta de Selección de Variables despliega la lista de variables disponibles. Usted puede hacer clic en cualquiera de estas variables para copiar el nombre de la variable en el componente de búsqueda. Cada uno de estas variables tiene el siguiente formato:

@nombreVariable: Descripción de variable.
(Ayuda para el tipo de variable: Instrucciones)

La ayuda para el tipo de variable puede ser una de las siguientes:

  • Escriba una palabra: Esto significa que la variable es de tipo texto simple.
  • Palabras: Esto significa que la variable es de tipo texto completo habilitado.
  • Escriba true/false: Esto significa que la variable es de tipo boolean.
  • Escriba un número: Esto significa que la variable es de tipo numérica.
  • Escriba una fecha: Esto significa que la variable es de tipo fecha.

Por ejemplo, un criterio de búsqueda que incluye variables sería "@nombre juan and @apellido perez" y esto buscaría todas las personas cuyo nombre es Juan y que apellidan Perez. Usted puedde utilizar paréntesis y las operaciones lógicas and/or (y/o) para especificar una búsqueda más compleja. Por ejemplo, usted puede especificar "(@nnombre juan or @nombre maria) and @apellido perez" para buscar todas las personas cuyo nombre es Juan o Maria y que apellide Perez.

Dependiendo del tipo de columna, la búsqueda tiene diferentes reglas:

Búsqueda numérica:

El operador de igualdad está implícito, así que para testear la igualdad simplemente se coloca el nombre de la variable seguido del valor que usted está buscando: "@id 100"

También puede utilizar los siguientes operadores en una búsqueda numérica:

  • > para buscar mayor que "@id > 5"
  • < para buscar menor que "@id < 10"
  • >= para buscar mayor o igual que "@salario >=10000"
  • <= para buscar menor o igual que "@salario <= 50000"
  • != para buscar inegualdad "@codigo != 5"
  • : para buscar un rango inclusivo "@codigo 5:10" buscará todos los registros con código de 5 a 10 inclusive
  • in() para buscar todos los valores en una lista "@codigo in(20, 45, 67)" buscará todos los registros con código 20, 45 o 67. No puede haber un espacio entre la palabra IN y el paréntesis que abre (i.e. @codigo in (1, 2, 3) no funcionará)

Búsqueda en fechas:

Debe utilizarse exactamente igual que la búsqueda numérica excepto que las fechas deben estar escritas en la notación internacional estándar, el cual es: AAAA-MM-DD, donde AAAA es el año en el calendario Gregoriano, MM es el mes del año entre 01 (Enero) y 12 (Diciembre), y DD es el día del mes entre 01 y 31. Por ejemplo, el cuarto día de Febrero del año 1995 se escribe en la notación estándar así: 1995-02-04.

El operador de igualdad es implícito, así que para buscar una fecha en particular solamente debe colocar el nombre de la variable seguido del valor de fecha que usted desea buscar: "@fechaRegistro 2001-6-1"

Usted puede utilizar los siguientes operadores para una búsqueda de fechas:

  • > busca las fechas mayores que "@fechaNacimiento > 1966-1-1"
  • < busca las fechas menores que "@fechaNacimiento < 2012-5-3"
  • >= busca las fechas mayores o iguales que "@fechaNacimiento >= 1966-1-1"
  • <= busca las fechas menores o iguales que "@fechaNacimiento <= 1966-1-1"
  • != busca las fechas diferentes de "@fechaRegistro != 2001-5-1"
  • : busca un rango inclusivo "@fechaRegistro 2011-1-1:2011-5-1" buscará todos los registros con una fecha de registro entre el 1ero de Enero al 1ero de Mayo de 2011 inclusive

Adicionalmente, usted puede utilizar funciones $year, $month y $day que se aplicarán a la variable de fecha. Por ejemplo, para buscar las personas cuyo cumpleaños es el 5 de Mayo se escribe: "$day(@fechaNacimiento) 5 and $month(@fechaNacimiento) 5".

Por supuesto, usted puede utilizar los operadores de comparación con estas funciones.

Búsqueda de texto simple:

Una búsqueda de texto simple busca textos usando dos expresiones muy simples:

  • Búsqueda simple de una palabra, o parte de una palabra, utilizando el operador opcional *. Por ejemplo "@nombre juan* and @apellido *erez"
  • Una búsqueda simple de un grupo de palabras en comillas dobles: "@nombre "Ana Maria" and @apellido Perez"

Búsqueda de texto completo:

Una búsqueda de texto completo es similar a la búsqueda simple, pero trata de ser un poco más inteligente para encontrar lo que usted está buscando.

En este tipo de búsqueda, cuando usted escribe "@tipoObjeto flor" el sistema buscará todas las formas de flor, incluyendo inflexiones. Por ello, usted puede encontrar registros con flor, flores, etc.

En este tipo de búsqueda usted tambien puede utilizar lo siguiente:

  • Todas las formas de inflexión de flor: @tipoObjeto flor
  • Todas las formas de inflexión de "flores florecen": @tipoObjeto "flores florecen"
  • Exactamente flores y formas de florecer: @tipoObjeto +flores and @tipoObjeto florecer
  • El texto flores está cerca del texto florecer: @tipoObjeto {flores florecer}

Búsqueda booleana:

Este es el tipo más simple de variable. Los únicos valores que se permiten aquí son (true significa verdadero): true (o 1) and false (ó 0)
